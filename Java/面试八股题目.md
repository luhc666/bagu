# 面试八股
1. 线程池是什么模式的体现，如何实现生产者和消费者的解耦的
	* 线程池是一种设计模式，通常被视为**池化模式（Pooling Pattern）**的一个具体应用。
	* 线程池可以有效地实现生产者-消费者模式，在这个模式中，生产者负责生成任务，消费者负责处理任务。通过使用线程池，生产者和消费者之间可以实现解耦。
2. 一个父线程能直接trycatch子线程的异常吗
在 Java 中，父线程不能直接捕获子线程抛出的异常。每个线程都有自己的异常处理机制，子线程中的异常只能在子线程内部被捕获，或者通过其他机制（如 Future、Thread.UncaughtExceptionHandler 等）被父线程知晓。
3. final关键字用于表示常量。它可以与变量，方法和类一起使用。任何实体（变量，方法或类）一旦被声明final后，只能分配一次。也就是：
	* final变量不能用另一个值重新初始化
	* final方法不能被重写
	* final类不能被继承
4. java中泛型底层是如何实现的：
类型擦除，也就是说java里面的泛型只存在于源代码里面，一旦经过编译之后，所有的泛型都会被擦除掉，全部被转换为 Object 类型，并在对元素进行访问和修改的时候，才会加上强制类型转换。这个强制类型转换是指从Object类转换过来。（所谓的裸类型指的是，ArrayList<Integer> 他的裸类型就是ArrayList）。也正是因为这样，如果我们在运行时判断ArrayList<String> 和ArrayList<Integer>会发现他们属于一个类型。
5. 类型擦除的缺点：
	* 使用类型擦除直接导致了对于原始的数据类型无法支持
	* 由于类型擦除的存在，Java 泛型无法在运行时创建泛型类型的实例。例如，我们无法使用 new T() 创建一个泛型类型 T 的实例，因为在类型擦除后，T 被转换为 Object 类型。这也导致了一些问题，例如无法创建泛型数组等。
	* 泛型类型参数不能使用 instanceof
	* 由于类型擦除的存在，Java 泛型无法在编译时区分不同类型的泛型方法，因此不能重载泛型方法。
6. 公平锁和非公平锁的优缺点：
公平锁：
	* 优点：所有的线程都能得到资源，不会饿死在队列中。
	* 缺点：吞吐量会下降很多，队列里面除了第一个线程，其他的线程都会阻塞，cpu唤醒阻塞线程的开销会很大。
非公平锁：
	* 优点：可以减少CPU唤醒线程的开销，整体的吞吐效率会高点，CPU也不必取唤醒所有线程，会减少唤起线程的数量。
	* 缺点：可能导致队列中间的线程一直获取不到锁或者长时间获取不到锁，导致饿死。
7. ConcurrentHashMap源码中的volatile有什么用：
	* 修饰Node中的val，保证值的可见性（一个线程修改后，其他线程立即可见）并避免脏读。
	* 修饰成员变量table和nextTable，在一个线程触发扩容的场景下，另外一个线程在读取时能够读到新链表
	* 修饰sizeCtl，确保链表的状态对所有线程可见。
